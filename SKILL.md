# 数模论文智能架构师

## 核心原则（必读）

生成论文前，遵守两条硬性原则：

### 原则一：数值单一真相源

论文中同一指标的数值**只能有一个版本**。若 WAPE=0.2881 出现在验证报告中，则摘要、正文表格、结论必须全部写 0.2881，不能有的写 0.288、有的写 0.2881。

**数值来源优先级：**
1. 独立验证报告（如 `verification_summary.md`）→ 最权威
2. 结果表 Excel（如 `结果表1.xlsx`）→ 行数/字段以此为准
3. 代码运行日志 → 次选
4. 早期阶段设计文档 → 仅供参考，不可直接引用

**关键做法：** 在生成或修改论文前，先向用户确认"以哪份文件为最终数值口径"。如果存在 `verification_summary.md` 或 `chapter_revision_checklist.md`，优先按其数值。

### 原则二：验证优先于成文

论文中的所有定量结论必须**可追溯到独立验证结果**。如果用户尚未运行独立验证，在论文中标注"数值待验证"，并在生成后提醒用户：

> 论文中数值来自代码直接输出，尚未经过独立验证。建议先运行 verification.py 产出验证报告，再据此统一修改论文数值。

如果用户已经运行了独立验证且结果为 PASS，**必须在摘要末尾和模型检验部分声明这一事实**：
```
综合独立验证结果为PASS，说明模型实现与结果输出具有较高可信度。
```

## 工作流

```
输入（题目+代码+笔记+结果） → 解析材料 → 完备性检查 → 数学形式化 → 数值口径确认 → LaTeX 生成
```

按顺序执行，材料不全在第二步主动追问。数值口径不清楚在第四步后追问。用户确认"先写后补"则标注 `[待补充]` 继续。

---

## 第一步：解析材料

读取用户提供的所有材料。代码逆向建模要点：

1. **目标函数** — 代码中 `minimize`、`maximize`、`loss`、`obj` 等关键字段
2. **约束条件** — 不等式约束、等式约束、边界条件、特殊节点限制
3. **算法参数** — 种群大小、交叉率、迭代次数、求解器设置（MIPGap、时间限制、warm start策略）
4. **输入输出变量** — 已知量、待求量、中间变量

| 材料类型 | 提取内容 |
|---------|---------|
| 题目描述/截图 | 赛题编号、子问题、已知数据、约束条件 |
| Python 代码 | 目标函数、约束条件、算法名称及参数、输入输出变量 |
| MATLAB 代码 | 同上，注意矩阵运算习惯 |
| 建模笔记 | 建模思路、假设、未完成的分析 |
| 运行结果/图表数据 | 定量数值、图表含义、对比结论 |
| 验证报告 | **所有最终数值以此为准** |

不认识的算法先搜索再写。

## 第二步：完备性检查

生成前检查以下要素是否齐全。**缺失主动追问**，不自行编造数据：

- [ ] 每个子问题有模型/求解结果？
- [ ] 有灵敏度分析数据？
- [ ] 随机/迭代算法有收敛性分析？
- [ ] 有模型检验（误差/残差/交叉验证/约束违反检查）？
- [ ] 有可视化图表数据？
- [ ] **有独立验证结果？** 若有，以验证报告数值为准

追问示例：
> 检测到 Q2 使用了 Gurobi MILP 求解，但未看到约束违反检查或独立验证结果。是否需要先运行验证再写论文？
> 目前缺少灵敏度分析数据，请提供参数扰动测试结果。

用户说"先写后补"，则标注 `[待补充]` 继续。

## 第三步：数学形式化

### 3.1 问题分析

从建模者视角拆解子问题关系（递进/并列），说明模型选择动机。配总体技术路线图。

### 3.2 模型假设

**必须用文字段落（不能用表格），每条带编号独立成段。** 假设不能空洞——避免写"数据准确可靠"而不交代具体怎么处理。正确写法：

```latex
\textbf{假设1：}题目给定的走货路由和历史货量数据在清洗后可靠。
对少量缺失设备节点（共5个），采用现有场地中位数配置补全，并在结论中承认该处理带来局部不确定性。

\textbf{假设2：}同一始发--目的流向在优化期内采用固定走货路径。
若附件表1中同一OD存在多条路由，默认选取第一条作为常用路径；
若历史货量中存在但路由表缺失（共279个流向），按直达路径处理。

\textbf{假设3：}混包约束采用强一致性充分条件建模。
同一$(i,j)$方向统一拆包点，保证不出现多拆包点冲突。此约束是题意的充分条件而非等价条件。

\textbf{假设4：}$\Gamma$-场景预算压力测试仅作为鲁棒性增强分析。
其借鉴预算不确定思想构造压力场景，并非严格的Bertsimas-Sim对偶鲁棒优化模型。
```

**假设写作规则：**
- 每条假设必须与后续模型直接相关——不写"仅考虑理想条件"一类套话
- 关键处理细节（如"5个缺失节点"、"279个无路由流向"、"6930条极端值"）直接写入假设
- 对方法性质的限定（如"这是压力测试不是严格鲁棒优化"）在假设中明确声明

### 3.3 符号说明

**紧跟模型假设之后、模型建立之前，作为正文第4章，绝对不能放附录。** 用三线表，只写主要全局变量。

```latex
\begin{table}[h]
\centering\caption{本文的主要符号说明}\label{tab:symbols}
\begin{tabular}{cll}\toprule
符号 & 含义 & 单位 \\\midrule
$Q_{rd}$ & 流向$r$在日期$d$的预测货量 & 件/天 \\
$G_i$ & 场地$i$现有机器格口数 & 个 \\
$x^M_{rs}$ & 0-1变量，流向$r$选择方案$s$且机器集包 & -- \\
$y_{ijk}$ & 0-1变量，在场地$i$经下一分拣流向$j$的包裹统一在$k$拆包 & -- \\
\bottomrule\end{tabular}
\end{table}
```

注意：
- 临时变量不写
- 即便在符号表中解释过，该符号在文中首次出现时仍需说明
- 三元组变量（如 $y_{ijk}$）的解释要准确到各维度含义

## 第四步：数值口径确认

**在生成 LaTeX 之前，先做数值口径对齐。** 这是本技能区别于普通 LaTeX 生成器的关键步骤。

### 4.1 列出所有将出现在论文中的定量结果

从解析的材料中提取所有定量数值，形成清单：

```
Q1: WAPE, MAPE, RMSE, MAE, 分层WAPE, 各层样本数
Q2: 机器/人工流向数, 目标值(如有), MIPGap, 约束违反数(格口/产能/混包/特殊节点)
Q3: 设备型号数量, 购置场地数, 人工场地数, 总年化成本, 重优化违反数
Gamma: 场景数, 固定增长最低可行率, Gamma基准可行率
结果表: 行数, 字段列表
```

### 4.2 确认数值来源

向用户确认：这些数值的最终来源是哪份文件？如果存在多个版本，以哪个为准？典型追问：

> 检测到 Q2 机器/人工流向在代码日志中为 5697/2424，但验证报告中为 3290/4831。请确认以哪个为准。

### 4.3 生成数值口径表

在论文生成前（或修改论文前），先整理一份"数值口径表"写入 paper.tex 顶部的注释块，确保所有位置引用同一数值。格式如下：

```latex
% === 数值口径（来源：verification_summary.md, 2026-05-15） ===
% Q1: WAPE=0.2881, MAPE=1.8124, RMSE=890.06, MAE=233.92
% Q1分层: large WAPE=0.2804(MAPE=0.4030), mid=0.3016(0.7984), small=0.5062(3.4244)
% Q2: 机器=3290, 人工=4831, 格口/产能/混包/U0误用 违反全0
% Q3: 3A+38D, 购置场地=41, 人工场地=88, 总年化=116,295,050, 重优化违反=0
% Gamma: 10场景, 固定增长最低可行率=94.90%, Gamma基准可行率=57.14%
% 结果表: 表1=56847行, 表2=18791行, 表3=双工作表
```

### 4.4 修改论文时使用 revision checklist 模式

当用户要求"根据新结果修改论文"，不要逐句追问"这里要改吗"。而是：

1. 先列出所有出现旧数值的位置（摘要/正文表格/结论/附录）
2. 生成一张 old→new 映射表
3. 用户确认后，逐章批量修改

示例映射表格式：
```
位置              | 旧值            | 新值
摘要 Q2           | 5697/2424       | 3290/4831
表 tab:q2_perf    | 5697/2424       | 3290/4831
表 tab:cost_sens  | 5697/2424       | 3290/4831
结论段            | 5697/2424       | 3290/4831
```

## 第五步：LaTeX 论文生成

### 5.1 论文结构

```
第一页：论文题目 + 摘要 + 关键词（同页）
第二页：目录
第三页起：正文（页码从1开始）
1. 问题重述 → 2. 问题分析 → 3. 模型假设 → 4. 符号说明
5~N. 各问题模型的建立与求解（每问题含：数据预处理、模型建立、求解、结果分析）
模型检验与评价 → 参考文献（GB/T 7714） → 附录
```

### 5.2 摘要规范（最重要）

**三段式结构：前言(3-5行背景) → 正文(每问题一段含结果) → 收尾(评价/创新/推广)。**

摘要必须呈现求解结果——用户明确要求时补充数值，不能只有方法描述没有数。

```latex
% 前言：背景 + 问题 + 整体方法
物流网络中的集包规则直接影响分拣格口占用与运营成本。本文围绕某物流企业
93个分拣中心的实际运营数据，针对未来货量预测、集包规则优化和设备购置决策
三个递进问题，构建"预测—优化—决策"一体化模型。

% 正文：每问题一段，必须含关键结果数值
\textbf{对于问题一，}建立LightGBM、Holt-Winters指数平滑和滚动均值的分层自适应
组合预测模型。按历史日均货量将流向划分为大、中、小三类，在验证集上分别优化
融合权重。整体WAPE为0.2881，分层融合主要提升模型鲁棒性，大流向WAPE=0.2804，
中流向WAPE=0.3016，小流向WAPE=0.5062。

\textbf{对于问题二，}建立含机器集包与人工集包双变量的混合整数线性规划模型。
通过候选方案生成、瓶颈评分和贪心warm start后调用Gurobi求解，得到3290个机器
集包流向和4831个人工集包流向，MIPGap为0.33\%，格口违反0，产能违反0，混包冲突0。

\textbf{对于问题三，}在Q2最优规则基础上将货量放大至1.2倍，逐场地计算缺口并
求解设备选型整数规划，扩容后重新优化集包规则。最终购置3台A型和38台D型设备，
共覆盖41个购置场地和88个人工场地，总年化成本116,295,050元，重优化后约束
违反数为0。$\Gamma$-场景压力测试共10个场景，固定增长场景最低可行率为94.90\%，
$\Gamma$预算场景基准可行率为57.14\%。

% 收尾：创新点 + 验证 + 推广
本文主要创新体现在自适应分层融合预测、双变量MILP集包优化和$\Gamma$场景压力
测试三个方面，对物流网络集包优化与产能规划具有推广价值。综合独立验证结果为
PASS，说明模型实现与结果输出具有较高可信度。
```

**创新点表述诚实性：**
- 画了热力图 ≠ "网络拓扑可视化诊断"为核心创新。降为辅助亮点
- 场景压力测试 ≠ "严格鲁棒优化"。写清楚是"固定方案压力测试"
- 预测精度 "显著优于基线" → "分层融合在大中流向上略优或不劣于单一模型"
- 诚实限定比过度声称更能赢得评委会信任

### 5.3 问题重述

用自己的话改写题目，不复制原文。说明情景、列出关键数据（包含具体数字：如"共1,151,312条记录、8,121个流向"）、提出明确问题。半页到一页。

### 5.4 问题分析

承上启下，每问题一小节，简明扼要不放结论。配总体技术路线图。对于递进结构的问题（Q1输出是Q2输入），明确说明依赖关系。

### 5.5 模型建立与求解（核心！）

每个问题的章节统一包含四部分：

#### (a) 数据预处理

描述数据来源、清洗逻辑、关键数字。用文字说明，不是只列公式。

```latex
首先对历史货量数据进行清洗。原始数据共1,151,312条记录、8,121个OD流向，
覆盖2025年12月1日至2026年5月10日的161天。清洗过程中识别出6,930条极端
高值样本，在建模前统一处理而非直接删除——物流场景中的尖峰可能来自促销或
节假日，简单剔除会低估容量需求。另有279个有货量流向在路由表中无对应路径，
构造直达路径补齐。5个设备信息缺失场地用同类场地中位数补全。
```

#### (b) 模型建立

**不只是列公式，必须有充分的文字说明：**

1. **建模动机** — 为什么选这个模型？问题本质是什么？
2. **逐步推导** — 从原理出发，解释每一步的物理/数学含义
3. **公式前后有文字** — 前说明为什么引入，后解释各部分含义
4. **约束解释** — 每个约束条件的实际含义
5. **与代码一致** — 文中模型必须与代码逻辑对应

示例：
```latex
为同时考虑机器集包和人工集包两种方式的成本差异，引入双变量$x^M_{rs}$和
$x^H_{rs}$分别表示流向$r$选择方案$s$且采用机器/人工集包。机器集包成本包含
单位处理成本$c_M$、瓶颈格口稀缺性惩罚$\eta_i$和拆包惩罚$\lambda$：

\begin{equation}
C^M_{rs}=\sum_{i\in N}A_{rsi}
\left(c_MQ_r+\eta_i\mathrm{slot}_{rsi}\right)
+\lambda(D_{rs}-1)Q_r,
\label{eq:cost_M}
\end{equation}

其中$A_{rsi}$为方案$s$在场地$i$的处理指示，$D_{rs}$为拆包次数，
$\mathrm{slot}_{rsi}=A_{rsi}\lceil Q_r/a_i\rceil$为占用格口数。
\eqref{eq:cost_M} 第一项为处理成本与格口惩罚，第二项为多拆包惩罚——
拆包次数越多，中转环节越多，成本越高。
```

**关键原则：**
- 每个公式前有引入文字，后有解释文字
- 公式编号用 `\label{eq:xxx}`，后文用 `\eqref{eq:xxx}` 引用
- 模型必须与代码逻辑一致——生成后对照代码验证

#### (c) 模型求解

描述算法步骤、参数设置。复杂算法配伪代码或流程图。

```latex
\begin{algorithm}[h]
\caption{集包规则优化流程}\label{alg:q2}
\begin{algorithmic}[1]
\State 根据路径$P_r$为每个流向生成候选方案$S_r$
\State 计算方案成本、格口占用、产能占用和拆包次数
\State 贪心构造初始可行解，写入warm start
\State 建立MILP模型并调用Gurobi求解
\State 输出最优集包规则、机器/人工分配和约束检查结果
\end{algorithmic}
\end{algorithm}
```

**伪代码必须放在 `algorithmic` 环境内。** 使用 `\For{...}...\EndFor`、`\To`（不是 `\TO`）、`\State`。禁止在正文或 `equation` 里直接用这些宏。

Gurobi/CPLEX 等商业求解器的性能描述要保守："在设定时间限制内快速收敛并得到高质量可行解"，不将单次日志中的极短运行时间过度解释为普遍性能。

#### (d) 结果分析

**必须有表格 + 文字解读 + 图表引用。** 不是只放数字。

```latex
\begin{table}[h]
\centering\caption{Q2 MILP模型求解结果}\label{tab:q2_perf}
\begin{tabular}{lc}\toprule
指标 & 数值 \\\midrule
二进制变量数量 & 约55,000 \\
MIPGap & 0.33\% \\
机器集包流向 & 3,290 \\
人工集包流向 & 4,831 \\
格口容量违反 & 0 \\
机器产能违反 & 0 \\
混包一致性冲突 & 0 \\
\bottomrule\end{tabular}
\end{table}

由表\ref{tab:q2_perf}可知，求解完成后对全部5类约束进行独立验证，
均未发现违反。当前Q2已通过独立验证，综合状态为PASS。
```

结果分析与代码输出和结果表必须一致：
- 摘要中数值 = 正文表格数值 = 结论中数值 = 结果表中数值
- 结果表行数/字段列表必须与附录C中的描述一致
- 同一指标不存在"0.288"和"0.2881"两种写法

### 5.6 模型检验

独立验证通过的论文，模型检验段应至少包含：
- Q1 预测精度验证（WAPE、MAPE、RMSE，可附带分层指标）
- Q2 约束可行性验证（每类约束的违反数，0 violations）
- Q3 设备方案验证（重优化后违反数、成本口径）
- 压力测试结论
- 综合PASS声明

如果验证失败或有已知问题（如蒙特卡洛模块bug），诚实说明而非隐藏。

### 5.7 模型评价

**优点：** 每条优点对应一个具体的方法创新或工程贡献，不写"模型合理、结果可靠"等空话。可补充一句"当前结果已通过独立验证，综合状态为PASS"。

**缺点（按重要性排序）：**
1. 最突出的精度/可行性不足（如"小流向预测误差仍较高 WAPE=0.5062"）
2. 方法论限定（如"Gamma分析属于压力测试而非严格鲁棒优化"）
3. 数据局限（如"缺失设备数据用中位数补全带来局部不确定性"）
4. 已知bug（如"蒙特卡洛模块仍有待修复"）

**诚实比完美更重要。** 评委会看得出哪些是真实问题、哪些是搪塞。

### 5.8 结果表说明（附录）

附录中的结果表说明必须与**实际输出的 Excel 文件**完全一致。生成前去确认：

- [ ] 结果表1：行数？字段列表？
- [ ] 结果表2：行数？字段列表？有没有"集包方式"列？
- [ ] 结果表3：单表还是多工作表？有没有"人工人数"列？

不准确的常见错误：
- 结果表2字段写了"集包方式"但实际不存在该列
- 结果表3写了"人工人数"字段但实际不存在
- 行数写"约50000"而实际是56847

## 图表规范

### 图表插入

```latex
\begin{figure}[h]
\centering\includegraphics[width=0.85\textwidth]{figures/文件名.png}
\caption{图表标题}\label{fig:标签}
\end{figure}
```

图表标题居中，全文连续编号，正文中必须引用（"如图X所示"、"表X表明"）。

### 图表与正文一致性

**图表中显示的数据必须与正文数值一致。** 典型不一致：
- F6 饼图显示 6A+49D，但正文改为 3A+38D → 图需重新生成
- F12 成本拆分显示 46,456,600 元，但正文改为 116,295,050 元 → 图需重新生成

如果用户提供的图显示旧版数据，提醒重新生成。若无法重新生成，在正文中标注"图X显示早期方案，最新结果以表X为准"。

## 自检清单

生成完成后逐项检查：

**格式层面：**
- [ ] 一级标题居中，二三级左对齐
- [ ] 题目三号黑体，正文小四宋体
- [ ] 摘要与题目同页，关键词左对齐
- [ ] 目录第二页，正文第三页起，页码从1开始脚注居中
- [ ] 公式编号 (章节.序号)，图表全文连续编号
- [ ] 伪代码全在 algorithmic 环境，用 `\To` 非 `\TO`
- [ ] 模型假设文字段落（非表格），符号说明在模型之前（非附录）
- [ ] TOC 包含所有附录条目，`\addcontentsline` 文本与 `\section*` 一致

**内容层面：**
- [ ] 每问题章节含：数据预处理 + 模型文字说明 + 求解文字 + 结果分析 + 图表
- [ ] 每个公式前后有文字（前引入、后解释）
- [ ] 模型与代码逻辑一致
- [ ] 参考文献 GB/T 7714 格式，正文有 `\cite` 引用

**数值一致性（关键！）：**
- [ ] 摘要数值 = 正文表格数值 = 结论数值 = 结果表数值
- [ ] 数值来源唯一（验证报告优先），不存在 0.288/0.2881 混用
- [ ] 结果表行数与附录C描述一致
- [ ] 结果表字段列表与实际 Excel 文件一致
- [ ] 图表数据与正文新数值一致（旧图已标注或已提醒重新生成）

**诚实性：**
- [ ] 创新点不过度声称（热力图不是核心创新，压力测试不是严格鲁棒优化）
- [ ] Q1 表述用"稳定/适配/不劣于"，不用"显著优于"
- [ ] 已知bug（如蒙特卡洛）在不足部分诚实说明
- [ ] 求解器性能描述保守（不将单词运行时间过度解释）
