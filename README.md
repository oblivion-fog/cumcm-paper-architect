# 数模论文智能架构师 (CUMCM Paper Architect)

> 将零散的赛题描述、代码逻辑和运行结果自动转化为规范的数学建模竞赛 LaTeX 论文。

## 概述

**cumcm-paper-architect** 是 Claude Code 的 Agent Skill，专为数学建模竞赛论文撰写场景设计。输入赛题材料（PDF/截图/代码/数据/笔记/结果），输出可直接用 `xelatex` 编译的 `.tex` 论文文件。

## 核心原则

1. **数值单一真相源** — 论文中同一指标的数值只能有一个版本。所有定量结论必须追溯到独立验证报告或结果表，不能摘要写0.288、表格写0.2881。
2. **验证优先于成文** — 论文数值以独立验证结果为准。验证PASS必须在摘要和模型检验中声明。

## 核心流程

```
输入（题目 + 代码 + 笔记 + 结果）
  → 解析材料（逆向建模、提取目标函数/约束/算法参数）
  → 完备性检查（缺模型？缺检验？缺图表？主动追问）
  → 数学形式化（问题分析 → 模型假设 → 符号说明）
  → 数值口径确认（列出所有定量结果 → 确认来源 → 生成口径表）
  → LaTeX 生成（完整的论文结构，所有数值统一）
```

## 功能特性

- **代码逆向建模** — 从 Python/MATLAB 代码自动推导目标函数、约束条件、算法参数和变量定义
- **完备性检查** — 自动检测缺失的子问题模型、灵敏度分析、收敛性分析、图表数据、独立验证，主动追问
- **数值口径管理** — 生成前先对齐所有定量数值来源，生成口径表注释写入 .tex 文件顶部。修改论文时使用 old→new 映射表逐章批量修改，防遗漏
- **结构化论文生成** — 严格遵循竞赛论文结构：摘要→问题重述→问题分析→模型假设→符号说明→模型建立与求解→模型检验→参考文献→附录
- **GB/T 7714 参考文献** — 自动生成标准参考文献条目
- **伪代码生成** — 从算法代码反向生成 `algorithmic` 环境伪代码
- **诚实性约束** — 创新点不过度声称（热力图≠核心创新、压力测试≠严格鲁棒优化）、模型假设具体化（"5个缺失节点用中位数补全"而非"数据准确可靠"）、已知bug诚实说明

## 输出规范

### 论文结构

```
第一页：论文题目 + 摘要 + 关键词（同页）
第二页：目录
第三页起：正文（页码从1开始）
├── 1. 问题重述
├── 2. 问题分析
├── 3. 模型假设（文字段落，非表格）
├── 4. 符号说明（三线表）
├── 5~N. 各问题模型的建立与求解
│   ├── 数据预处理（含关键数字）
│   ├── 模型建立（公式前后有文字）
│   ├── 模型求解（伪代码/流程图）
│   └── 结果分析（表格+解读+图表引用）
├── 模型检验与评价
├── 参考文献（GB/T 7714）
└── 附录
```

### 摘要规范

三段式结构：**前言(3-5行背景) → 正文(每问题一段含结果) → 收尾(评价/创新/推广)**。摘要必须呈现求解结果数值。

### 排版要求

- 一级标题居中，二三级左对齐
- 题目三号黑体，正文小四宋体单倍行距
- 摘要与题目同页，关键词左对齐
- 公式编号 `(章节.序号)`，图表连续编号
- 三线表，`booktabs` 宏包

### LaTeX 依赖

| 宏包 | 用途 |
|---|---|
| `ctex` | 中文支持 |
| `amsmath, amssymb, amsfonts` | 数学公式 |
| `algorithm, algpseudocode` | 伪代码 |
| `booktabs` | 三线表 |
| `graphicx` | 插图 |
| `cite` | 参考文献引用 |
| `listings` | 代码附录 |
| `enumitem, indentfirst` | 列表与缩进 |

## 使用方式

1. 在 Claude Code 中触发技能（提及"写论文"、"数模论文"、"LaTeX论文"、"MCM/ICM" 等关键词）
2. 提供材料：赛题 PDF/截图、Python/MATLAB 代码、建模笔记、Excel 数据、运行结果、验证报告（任意组合）
3. 技能自动执行：解析 → 检查 → 形式化 → 数值口径对齐 → 生成
4. 输出完整 `.tex` 文件，可直接用 `xelatex` 编译

## 修改论文工作流

当已有论文需要根据新验证结果修改时：

1. 列出所有出现旧数值的位置（摘要/正文表格/结论/附录）
2. 生成 old→new 映射表，用户确认
3. 逐章批量修改，先统一数值口径，再调整论证强度，最后清理细节

## 自检清单

### 格式层面
- 一级标题居中，二三级左对齐
- 题目三号黑体，正文小四宋体
- 摘要与题目同页，关键词左对齐
- 目录第二页，正文第三页起，页码从1开始
- 公式编号 (章节.序号)，图表全文连续编号
- 伪代码在 `algorithmic` 环境，用 `\To` 非 `\TO`
- 模型假设文字段落，符号说明在第4章
- TOC 包含所有附录条目

### 数值一致性
- 摘要数值 = 正文表格数值 = 结论数值 = 结果表数值
- 数值来源唯一（验证报告优先），不存在同一指标多版本
- 结果表行数/字段与附录描述一致
- 图表数据与正文新数值一致

### 诚实性
- 创新点不过度声称
- Q1 表述用"稳定/适配"不夸大
- 已知bug诚实说明
- 求解器性能描述保守
- Gamma 写"固定方案压力测试"非"严格鲁棒优化"
