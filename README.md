# 数模论文智能架构师 (CUMCM Paper Architect)

> 将零散的赛题描述、代码逻辑和运行结果自动转化为规范的数学建模竞赛 LaTeX 论文。

## 概述

**cumcm-paper-architect** 是 [Claude Code](https://claude.ai/code) 的 Agent Skill，专为数学建模竞赛场景设计。输入赛题材料（PDF/截图/代码/数据/笔记/结果），自动输出可直接用 `xelatex` 编译的 `.tex` 论文文件。

## 核心流程

```
输入（题目 + 代码 + 笔记 + 结果）
  → 解析材料（逆向建模、提取目标函数/约束/算法参数）
  → 完备性检查（缺模型？缺检验？缺图表？主动追问）
  → 数学形式化（问题分析 → 模型假设 → 符号说明）
  → LaTeX 生成（完整的论文结构）
```

| **MCM/ICM（美赛）** | 美国大学生数学建模竞赛 |

## 功能特性

- **代码逆向建模** — 从 Python/MATLAB 代码自动推导目标函数、约束条件、算法参数和变量定义
- **完备性检查** — 自动检测缺失的子问题模型、灵敏度分析、收敛性分析、图表数据，主动追问
- **结构化论文生成** — 严格遵循竞赛论文结构：摘要→问题重述→问题分析→模型假设→符号说明→模型建立与求解→模型检验→参考文献→附录
- **GB/T 7714 参考文献格式** — 自动生成标准参考文献条目
- **伪代码生成** — 从算法代码反向生成 `algorithmic` 环境伪代码
- **图表占位符** — 无图表文件时自动插入 `[待插入]` 占位，不阻塞论文生成

## 输出规范

### 论文结构

```
第一页：论文题目 + 摘要 + 关键词
第二页：目录
第三页起：正文（页码从1开始）
├── 1. 问题重述
├── 2. 问题分析
├── 3. 模型假设
├── 4. 符号说明
├── 5~N. 各问题模型的建立与求解
│   ├── 数据预处理
│   ├── 模型建立
│   ├── 模型求解
│   └── 结果分析
├── 模型检验与评价
├── 参考文献
└── 附录
```

### 排版要求

- 一级标题居中，二三级左对齐
- 题目三号黑体，正文小四宋体单倍行距
- 摘要与题目同页，关键词左对齐
- 公式编号 `(章节.序号)`，图表连续编号
- 三线表，`booktabs` 宏包

### LaTeX 模板

使用 `\documentclass[12pt,a4paper]{article}`，依赖以下宏包：

| 宏包 | 用途 |
|---|---|
| `ctex` | 中文支持 |
| `amsmath, amssymb, amsfonts` | 数学公式 |
| `algorithm, algpseudocode` | 伪代码 |
| `booktabs` | 三线表 |
| `graphicx` | 插图 |
| `cite` | 参考文献引用 |
| `listings` | 代码附录 |

## 使用方式

1. 在 Claude Code 中触发技能（提及"写论文"、"数模论文"、"LaTeX论文"、"MCM/ICM" 等关键词）
2. 提供材料：赛题 PDF/截图、Python/MATLAB 代码、建模笔记、Excel 数据、运行结果（任意组合）
3. 技能自动执行：解析 → 检查 → 形式化 → 生成
4. 输出完整 `.tex` 文件，可直接用 `xelatex` 编译

## 安装

该技能已预置于 Claude Code skills 目录。确保 `.claude/settings.json` 中包含本技能引用即可。

## 自检清单

生成后自动检查：
- 标题层级（一级居中，二三级左对齐）
- 摘要与题目同页，关键词格式正确
- 目录、正文页码正确
- 公式图表编号与引用完整
- 伪代码在 `algorithmic` 环境中
- 模型假设为文字段落（非表格）
- 符号说明位于模型之前（非附录）
- 每个问题含完整的 4 个子节
- 模型逻辑与代码一致
- 参考文献 GB/T 7714 格式
- TOC 包含所有附录条目
